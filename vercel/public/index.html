欢迎访问 NetGit 代理服务

这是一个由 Vercel 托管的 NetGit 服务入口 ()。 本服务旨在提供灵活、快速的代理能力，帮助您轻松访问 GitHub、Docker Hub 等资源。

🚀 NetGit 代理规则列表

以下是该服务当前支持的代理规则，您可以方便地使用它们来访问特定的网络资源。

GitHub 资源代理

  • 代理路径: /ghproxy/
  • 代理模式: URL 参数
  • 使用示例: 
    /ghproxy/github.com/louislam/uptime-kuma/archive/refs/tags/2.0.2.zip

Docker Hub 镜像代理

  • 代理路径: /dockerproxy/
  • 代理模式: 路径映射
  • 目标源站: https://registry-1.docker.io
  • 使用示例: 
    /dockerproxy/v2/

Catbox.moe 文件代理

  • 代理路径: /catbox/
  • 代理模式: 路径映射
  • 目标源站: https://files.catbox.moe
  • 使用示例: 
    /catbox/yqh1it.png

Koyeb WebSSH 透明代理 (Host 模式)

  • 代理路径: /ssh/
  • 代理模式: Host 模式 (透明转发)
  • 目标源站: subsequent-ardelle-bbttca23-472bd3ef.koyeb.app
  • 使用示例: 
    /ssh/
重要提示：
  • URL 参数模式: 代理路径后必须跟上完整的、在白名单内的 URL。
  • 路径映射模式: 代理路径后直接跟上目标站点的文件路径。
  • Host 模式 (透明转发): 专为需要保留 Host 头等信息的应用设计。它会剥离代理路径，将剩余路径完整转发到目标域名（当然也可以像我一样用来来缩短域名）。
